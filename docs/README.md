# Maya脚本管理器

## 简介

Maya脚本管理器是一个功能强大的Maya工具，专为提高脚本管理效率而设计。它提供了直观的用户界面，让您能够轻松组织、管理和运行Python和MEL脚本，大幅提升工作流程效率。

### 核心功能

- **智能脚本管理** - 创建、编辑、组织和一键运行Python/MEL脚本
- **分组管理系统** - 将脚本按功能或项目分类整理，支持拖拽排序
- **拖放导入** - 直接拖放外部.py或.mel文件快速导入
- **回收站机制** - 安全删除脚本，支持恢复和永久删除
- **智能提示系统** - 鼠标悬停显示脚本功能说明，支持自动提取和手动编辑
- **配置管理** - 完整的配置导入/导出功能，支持备份和团队共享
- **文件夹同步** - 基于文件夹结构的自动扫描和同步机制
- **搜索功能** - 实时搜索脚本，快速定位所需工具
- **自定义按钮颜色** - 为不同脚本设置个性化颜色标识
- **布局选择** - 支持单列和双列按钮布局
- **内置脚本编辑器** - 集成代码编辑器，支持语法高亮和拖放加载

### 兼容性

- **Maya版本**: 2021-2026 (完全兼容)
- **Qt框架**: 自动适配PySide2和PySide6
- **操作系统**: Windows、MacOS、Linux
- **编码支持**: UTF-8、GBK、Latin-1多编码兼容

## 安装指南

### 快速安装（推荐）

1. 下载脚本管理器完整包
2. 解压到任意位置（建议放在易于访问的目录）
3. 打开Maya
4. 将整个`scripts box`文件夹拖放到Maya视口中，或者直接拖放`scripts_box.py`文件
5. 脚本管理器将自动启动

### 手动安装

1. 将`scripts box`文件夹复制到Maya脚本目录：
   - **Windows**: `Documents\maya\scripts\`
   - **MacOS**: `~/Library/Preferences/Autodesk/maya/scripts/`
   - **Linux**: `~/maya/scripts/`

2. 在Maya脚本编辑器中运行：

```python
import sys
import os

# 添加脚本路径（如果需要）
scripts_path = r"path/to/scripts box"  # 替换为实际路径
if scripts_path not in sys.path:
    sys.path.append(scripts_path)

# 启动脚本管理器
import scripts_box
scripts_box.show_scripts_box()
```

### 创建工具架按钮（可选）

为了方便使用，可以在Maya工具架中创建快捷按钮：

```python
import maya.cmds as cmds

# 创建工具架按钮
cmds.shelfButton(
    command="""
import sys
import os
scripts_path = r"path/to/scripts box"  # 替换为实际路径

    sys.path.append(scripts_path)
import scripts_box
scripts_box.show_scripts_box()
""",
    image="pythonFamily.png",

    annotation="打开Maya脚本管理器",
    imageOverlayLabel="SM"
)
```

## 使用指南

### 界面布局

脚本管理器采用现代化的分栏布局：
- **左侧导航栏**：显示所有分组按钮、新建分组按钮和回收站按钮
- **顶部工具栏**：包含新建脚本、刷新、搜索、设置和帮助按钮
- **主工作区**：显示当前选中分组的脚本按钮
- **搜索区域**：实时搜索结果显示

### 基本操作

#### 脚本管理

- **创建脚本**：点击顶部工具栏的"新建脚本"按钮，选择Python或MEL类型
- **运行脚本**：直接点击脚本按钮即可执行
- **编辑脚本**：右键点击脚本按钮 → "编辑脚本"（使用内置编辑器）
- **删除脚本**：右键点击脚本按钮 → "删除"（移入回收站）
- **修改提示信息**：右键点击脚本按钮 → "编辑提示信息"
- **自定义颜色**：右键点击脚本按钮 → "设置按钮颜色"
- **打开文件夹**：右键点击脚本按钮 → "打开文件夹"

#### 分组管理

- **创建分组**：点击左侧导航栏底部的"+"按钮
- **重命名分组**：右键点击分组按钮 → "重命名"
- **删除分组**：右键点击分组按钮 → "删除"（需确认）
- **移动脚本**：右键点击脚本按钮 → "移动到分组" → 选择目标分组
- **拖拽排序**：直接拖拽分组按钮调整顺序

#### 搜索功能

- **实时搜索**：在顶部搜索框输入关键词，实时显示匹配结果
- **搜索结果**：显示脚本名称、类型和所属分组
- **清除搜索**：清空搜索框或点击清除按钮返回正常视图

#### 导入与导出

- **拖放导入**：直接将.py或.mel文件拖放到窗口中
- **批量导入**：可同时拖放多个文件
- **导出配置**：点击"设置" → "导出配置"（可选择是否包含脚本文件）
- **导入配置**：点击"设置" → "导入配置"（可选择仅配置或包含脚本）

### 高级功能

#### 文件夹同步管理

脚本管理器基于文件夹结构自动管理脚本：

1. **目录结构**：
   ```
   scripts box/
   ├── scripts_box.py          # 主程序
   ├── tool/                   # 工具目录
   │   ├── 分组1/              # 每个子文件夹对应一个分组
   │   │   ├── script1.py
   │   │   └── script2.mel
   │   ├── 分组2/
   │   │   └── script3.py
   │   └── ...
   └── config.json            # 配置文件
   ```

2. **手动管理**：
   - 直接在`tool`文件夹中创建/删除子文件夹
   - 将脚本文件放入对应分组文件夹
   - 在Maya中点击"刷新"按钮同步更改

3. **自动扫描**：
   - 启动时自动扫描`tool`目录
   - 支持UTF-8、GBK、Latin-1多种编码
   - 自动识别Python(.py)和MEL(.mel)脚本

#### 回收站系统

- **安全删除**：点击左侧"回收站"按钮查看已删除脚本
- **脚本恢复**：在回收站中点击"恢复"按钮恢复到原分组
- **永久删除**：在回收站中点击"删除"按钮彻底删除
- **批量清理**：点击"清空回收站"按钮清除所有已删除脚本
- **智能提示**：显示删除时间和原始分组信息

#### 内置脚本编辑器

- **语法高亮**：支持Python和MEL语法着色
- **拖放加载**：直接拖放文件到编辑器加载内容
- **自动识别**：根据文件扩展名自动设置语言模式
- **实时编辑**：修改后自动保存到对应脚本文件
- **错误处理**：编码问题自动处理和转换

#### 自定义设置

- **按钮布局**：在设置中选择单列或双列布局
- **按钮颜色**：为每个脚本设置个性化颜色标识
- **提示管理**：支持自动提取注释或手动编辑提示信息
- **分组排序**：通过拖拽调整分组显示顺序
- **配置备份**：定期导出配置文件进行备份

## 常见问题与解决方案

### 安装和启动问题

**Q: 拖放脚本后没有反应？**
- 确保拖放的是`scripts_box.py`文件或整个`scripts box`文件夹
- 检查Maya脚本编辑器是否有错误信息
- 尝试手动运行：`exec(open(r'path/to/scripts_box.py').read())`

**Q: 提示找不到模块？**
- 确保脚本路径已添加到`sys.path`
- 检查文件路径是否正确，注意路径分隔符
- 重启Maya后重试

### 脚本执行问题

**Q: 脚本无法运行？**
- 检查脚本语法是否正确
- 查看Maya脚本编辑器中的详细错误信息
- 确认Maya版本与脚本兼容性
- 检查脚本文件编码（支持UTF-8、GBK、Latin-1）

**Q: MEL脚本执行失败？**
- 确保MEL脚本语法正确
- 检查是否存在与文件名同名的MEL过程
- 尝试在Maya MEL命令行中手动执行

### 文件管理问题

**Q: 脚本丢失或找不到？**
- 首先检查回收站是否有被误删的脚本
- 点击"刷新"按钮重新扫描工具目录
- 检查`tool`文件夹中的文件是否存在
- 查看是否被移动到其他分组

**Q: 配置导入失败？**
- 确保配置文件格式正确（JSON格式）
- 检查配置文件是否包含必要的字段
- 如果导入脚本文件，确保存在对应的`tool`文件夹

### 界面显示问题

**Q: 界面显示异常或崩溃？**
- 重启Maya并重新打开脚本管理器
- 检查Maya版本兼容性（支持2021-2026）
- 删除`config.json`文件重置配置
- 更新到最新版本的脚本管理器
